2.1 Review the ongoing use of IT tools and change approach as needed

So far I’m happy with the tools we as a class have been using. Although prior to this course, I had only used Sublime for writing HTML, VS Code has proven immensely useful for typing in a Python and HTML format. It also allows you to download add-ons and change themes, giving me the opportunity to understand the errors I am making more clearly, and reading the code easier due to the colour theme.

Although overall I was happy with VS Code and the languages taught, using only HTML and CSS for our website design was difficult. My original approach to the website involved trying to use pure CSS to get the portions of large text to slide up under the question title. The most efficient way to code this would be via additional languages such as Javascript, and as I do not know this language yet, I decided to change my approach and keep the text on show.

Trello has been a very helpful tool throughout the course. On the group project, we were able to keep track of and prioritise our tasks, getting the story down much quicker than anticipated, which left us with plenty of time to finish our code. For the individual project, I kept using Trello to keep track of the questions and answers yet to tackle, as well as the general website building checklist. Having it in a Trello board also makes it an effective backup if anything ever happens to my text documents, which I use to start with typing up my thoughts before transferring them to VS Code or Trello.

Draw.io was a useful tool for visualising my thought process during both game design and website projects. During game design we were able to utilise the flowchart created via draw.io to create our story board and follow it when creating our code, without confusing ourselves on what functions should deliver what result. During the web design project, I was able to visualise my thoughts for the pathing of the website. Using the diagram both for and in the website pages. However, after reviewing all of the questions set for the written work, I decided to change my approach to the website layout, choosing to instead create drop down links for the navigation bar for each separate outcome in Unit 1.

Other tools used throughout the course were Slack, Zoom and GitHub. Slack was extremely effective as a communication tool, both for reaching out to team members, as well as getting my instructor to review my written work/code. Zoom was also helpful in communicating my bugs in the code, as I was able to share my screen for others to review the code with a different perspective. GitHub was helpful in being able to find open source code, and to understand how others have chosen to write their code in similar circumstances.

Overall, I am very confident with the tools I have used, but deeper learning and further research would make these tools even more helpful in the future.

2.3 Assess the strengths and weaknesses of your final project

So far I'm relatively happy with the website I designed. The current version of the website is very basic, but this is due to the fact that I have only been learning HTML and CSS for 3 days, and at that, still a very basic understanding of both languages. The layout is simple, with a dark background and readable font in shades of white. I was able to incorporate :hover functionality in blue and yellow, and after testing this colour scheme on a friend with a rare version of deuteranopia colour vision, and he was still able to differentiate the hover colours from the original white shades. Although I realise this is not a full-proof test for all colourblind people, for a project of this type, I still consider this a strength for my website.

However, accessibility as a whole, can be improved on this website. I have provided alt text for all images used, but I have yet to implement text-to-speech functionality on my website. As I only know very basic HTML and CSS, it is not something I was able to include, and if a person with vision impairment were to use the website as is, they would not be able to access the information on my website freely. This is definitely a weakness I hope to amed in the future.

Another strength of the website is the simplicity of structure. The each web page follows the structure of:

  1. Social media links
  2. Main heading
  3. Navigation bar
  4. Code nation project logo
  5. Heading 2 for outlining outcome objective
  6. Headings 3 for outlining the question
  7. Paragraphs to answer above question
  8. References with links
  9. Social media links

By using this structure on every page, I have achieved consistency and clarity for the website. As this was my original concept, I am glad I was able to bring it to fruition.

One weakness for my website would be the large amount of text that the reader will have to scrolls though. My original plan involved a slide up button next to each heading 3 question, so the reader had the option to hide the text and open it back up when needed. I wanted to achieve this through pure CSS code. Upon further research into this website function, I realised that the easier way would have been through the use of either Javascript or Bootstrap. Unfortunately, these languages are unfamiliar to me right now, and I didn’t want to just copy and paste the code in, without learning/understanding what it is I have done. Thus, I had to change my approach, and leave the text on show, and hoping the website user doesn’t mind scrolling through. The long body of text will remain, in my eyes, a weakness for this website design.

However, what I have managed to achieve is still impressive in my eyes. My plan is to return to this website once I have done more learning into other programming languages, to see if I may improve upon the current design.


2.4 Describe further improvements you can make to your project

As mentioned previously, accessibility is certainly something I would look to improve on this and future projects. My constraint on this occasion is lack of experience - had I had more understanding of how to utilise javascript or bootstrap to enable a text-to-speech function, it would have made the website a lot more accessible for users with vision impairment.

Another function I was not able to implement due to my lack of knowledge, was the slide up and slide down functions for the large bodies of text on the outcome 1, 2 and 3 pages. An improvement would have been to include a button next to every question heading, which upon pressing, reveals hidden text, and once revealed, the same button would also be able to hide the text once more, under the question headline. This was one of my biggest frustrations for this website, as it makes it a lot more difficult to digest such a large piece of text. I know that I prefer to compartmentalise large amounts of information in order to take it all in, and I’m concerned that this could impact ease-of-use for other such learners.

Therefore, in order to make further improvements to my website, as well as any future digital projects, I need to further improve upon my own experience and knowledge of programming languages. Javascript in particular, due to its universality and popularity, would be the programming language I focus on in the future.
